- 后端订单创建逻辑改为以后端数据库商品为准：
  - 校验摊位必须已审核释放。
  - 校验商品必须属于当前摊位。

### 模块 3/4：商家端状态与管理后台四级审核规则

- 用户端“我的订单”列表已支持点击进入订单详情。

  - 上架商品时必须选择已释放摊位。
  - 后端校验摊位必须属于当前商家，且 `audit_status=approved`。
  - 新增商品默认 `on_sale`，可直接在用户端摊位详情展示。
  - 商品列表展示所属摊位。
- 已验证：
  - `test4` 的“老城豆花摊”尚未释放时，调用商家商品上架接口会被拒绝，提示只能给已释放且属于自己的摊位上架商品。
- 管理后台已有四个真实审核队列：
  - 入驻审核。
  - 资质审核。
  - 特殊身份/公益标签审核。
  - 摊位预约审批。
- 后端补强摊位预约审批规则：
  - 必须先通过入驻审核。
  - 必须完成资质审核，不能有待审或驳回资质，且至少有一条已通过资质。
  - 必须完成公益/特殊身份审核，不能有待审或驳回申请，且至少有一条已通过申请。
  - 满足以上条件后，摊位预约通过才会释放到用户端。
- 已验证：
  - 对仍有待审资质的摊主直接审批摊位预约，会被后端拒绝，并提示先完成资质审核。
- 管理后台数据总览已接入真实 `/admin/dashboard/overview`：
  - 商家数、用户数、摊位数、订单数、商品数、投诉数来自后端真实统计。
  - 管理端构建已通过。

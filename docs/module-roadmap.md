# 模块开发路线

1. 用户登录、角色权限、JWT。
2. 商家入驻与资质审核。
3. 经营区域、摊位、预约审批。
4. 商品与媒资管理。
5. 摊位搜索、详情、地图导航 mock。
6. 预约订单与订单状态流转。
7. 评价、关注、收藏、订阅。
8. 投诉举报与监管处理闭环。
9. 政策公告、通知、数据看板。
10. 支付、短信、对象存储、地图真实接口接入。
11. 推荐算法、智能订单 Agent 等增强功能。

## 前端优先开发顺序

1. H5 响应式设计基底：移动端优先，同时支持 PC 浏览器。
2. 用户端：首页、摊位列表、摊位详情、订单、我的。
3. 商家端：商家开店、商家工作台、资质管理、摊位/出摊、订单处理。
4. 管理后台：登录、数据总览、商家审核、商品管理、投诉/订单列表。
5. 把原型中的次级页面逐步补齐，并持续沉淀通用组件和 API 类型。

详细说明见 `docs/frontend-development.md`。

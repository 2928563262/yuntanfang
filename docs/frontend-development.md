# 前端开发说明

## 当前输入

原型来源：

- `D:/OneDrive/Desktop/云摊坊-用户端.zip`
- `D:/OneDrive/Desktop/云摊坊-商家端.zip`
- `D:/OneDrive/Desktop/云摊坊-管理后台.zip`

识别结论：

- 用户端 PDF 原型约 105 页，主要是移动端 H5。
- 商家端 PDF 原型约 136 页，主要是移动端 H5。
- 管理后台 PDF 原型约 179 页，主要是桌面端后台。
- 移动端原型比例约为 375x812，但实现时必须同时支持手机浏览器和 PC 浏览器。

## 硬性要求

- 不做微信小程序，不新增小程序目录。
- 用户端和商家端都放在 `frontend/apps/h5`，移动端优先，同时适配 PC。
- 管理后台放在 `frontend/apps/admin`，以 PC 使用为主，同时保证窄屏可用。
- 前端页面先使用 mock 数据和 API 封装，不阻塞后端业务表完整实现。
- 页面应优先还原原型中的信息结构、导航层级、主要业务入口和状态表达，不追求一次性补齐所有 400+ 页面。
- 设计、业务顺序、接口约定要写入文档，便于后续 AI 接手。

## 响应式原则

H5 用户端和商家端：

- 手机端：单列信息流、底部 Tab、顶部导航、卡片式业务入口。
- PC 端：居中应用壳，主内容最大宽度控制在 1180px；首页、列表、工作台等页面使用双列或三列网格增强信息密度。
- 页面元素不能依赖固定 375px 宽度；使用 `minmax()`、`clamp()`、`max-width`、`grid`、`flex-wrap`。
- 业务按钮、状态标签、列表项在 PC 端不能被拉伸到难以阅读。

管理后台：

- 桌面端使用侧边栏、顶部栏、面包屑、表格、筛选区。
- 窄屏下侧边栏和内容区不能重叠，后续可演进为抽屉菜单。

## 开发顺序

### 阶段 1：前端基础与核心入口

1. 建立 H5 响应式设计基底：颜色、字号、间距、卡片、状态标签、列表、底部 Tab、PC 应用壳。
2. 用户端核心页面：首页、摊位列表、摊位详情、订单、我的。
3. 商家端核心页面：商家开店、商家工作台、资质管理、摊位/出摊、订单处理、我的。
4. 管理后台基础布局继续完善：登录、数据总览、商家审核、商品管理、订单/投诉列表。

### 阶段 2：用户端主流程

1. 登录/注册。
2. 附近摊位、搜索筛选、特色专区。
3. 摊位详情、商品、评价、导航 mock。
4. 预约下单、订单状态、评价。
5. 收藏、关注、订阅、投诉。

### 阶段 3：商家端主流程

1. 入驻备案表单。
2. 资质材料上传与特殊身份/公益标签。
3. 合法摊位查询、预约、出摊日历、经营打卡。
4. 商品上架、媒资、摊主故事、营销活动。
5. 订单处理、评价回复、经营数据。

### 阶段 4：管理后台主流程

1. RBAC 菜单和权限占位。
2. 商家入驻、资质、特殊身份审核。
3. 区域、摊位、预约审批。
4. 商品内容审核、评价管理。
5. 投诉工单、违规记录、超时告警。
6. 政策公告、通知、数据看板。

### 阶段 5：真实服务接入

1. 地图真实 Web API。
2. 支付、退款、对账。
3. 短信、对象存储、内容安全。
4. 推荐算法和智能订单 Agent。

## 当前实现策略

- 先在 `frontend/apps/h5` 建立用户/商家共用响应式组件风格。
- 路由保持与 `README.md` 和后端 API 骨架一致。
- mock 数据就近放在页面或后续抽到 `packages/shared`，接口调用通过 `packages/api`。
- 每个阶段完成后必须运行 `corepack pnpm -r --filter './packages/**' --filter './apps/**' build`。

# submit_review

## 功能

帮助用户根据订单提交评价。

## 触发语

- "给上一单好评"
- "帮我评价订单 1002，服务很好"
- "这单体验一般，给 3 星"

## 参数

```json
{
  "orderId": 1002,
  "rating": 5,
  "content": "服务很好，取餐很快。"
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `orderId` | number | 条件必填 | 订单 ID；用户说“上一单/最近一单/这单/这个订单”时后端可使用最近可评价订单 |
| `rating` | number | 必填 | 1-5 星；也可由“好评/中评/差评/一般”等明确评分信号转换 |
| `content` | string | 否 | 评价内容 |

必须有订单指代，且必须有明确评分信号。只说“我要评价”时必须先追问订单；只说“给上一单评价”时必须追问评分。

系统默认值：

- 用户明确说“上一单/最近一单/这单/这个订单”但未给订单号时，后端使用最近可评价订单 `1002`。
- 用户只给评分时，后端按评分生成匹配内容：低分为改进反馈，中评为一般反馈，高分为正向反馈。
- 用户说“好评/五星/五分”时后端可生成 5 星；说“中评/一般”时生成 3 星；说“差评/一星/不好/很差”时生成 1 星。

禁止行为：

- 不要在用户没有订单指代时默认使用最近订单。
- 不要把“我要评价”直接变成 5 星好评。
- 用户只说“服务很好、取餐很快”但没有星级或好评等评分信号时，必须追问几星。

## 参数不足响应

追问文案：

缺订单指代：

`你想评价哪个订单？可以说订单号，或明确说上一单、最近一单。`

缺评分信号：

`你想给几星？可以说 1-5 星，或明确说好评、中评、差评。`

## 响应 action

```json
{
  "type": "submit_review",
  "label": "确认发布评价",
  "route": "/my-reviews",
  "payload": {
    "reviewId": 185702,
    "orderId": 1002,
    "rating": 5,
    "content": "服务很好，取餐很快。",
    "status": "已发布"
  }
}
```

## 前端行为

用户确认后写入评价 store，并跳转 `/my-reviews`。

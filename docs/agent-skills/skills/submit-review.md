# submit_review

## 功能

帮助用户根据订单提交评价。

## 触发语

- "帮我评价订单 1002，服务很好"
- "这单体验一般，给 3 星"

## 参数

```json
{
  "orderId": 1002,
  "rating": 5,
  "content": "服务很好，取餐很快。"
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `orderId` | number | 必填 | 订单 ID，必须来自用户明确给出的订单号或前端上下文中的真实订单 |
| `rating` | number | 必填 | 1-5 星；也可由“好评/中评/差评/一般”等明确评分信号转换 |
| `content` | string | 否 | 评价内容 |

必须有具体订单号，且必须有明确评分信号。只说“我要评价”时必须先追问订单；只说“给上一单评价”时必须追问具体订单号。

系统默认值：

- 用户只给评分时，后端按评分生成匹配内容：低分为改进反馈，中评为一般反馈，高分为正向反馈。
- 用户说“好评/五星/五分”时后端可生成 5 星；说“中评/一般”时生成 3 星；说“差评/一星/不好/很差”时生成 1 星。

禁止行为：

- 不要在用户没有订单号时默认使用最近订单。
- 不要把“我要评价”直接变成 5 星好评。
- 用户只说“服务很好、取餐很快”但没有星级或好评等评分信号时，必须追问几星。

## 参数不足响应

追问文案：

缺订单指代：

`你想评价哪个订单？请告诉我具体订单号。`

缺评分信号：

`你想给几星？可以说 1-5 星，或明确说好评、中评、差评。`

## 响应 action

```json
{
  "type": "submit_review",
  "label": "确认发布评价",
  "route": "/my-reviews",
  "payload": {
    "orderId": 1002,
    "rating": 5,
    "content": "服务很好，取餐很快。",
    "status": "待确认"
  }
}
```

## 前端行为

用户确认后调用 `POST /api/orders/{orderId}/reviews`，请求体包含 `rating` 和 `content`。成功后评价写入数据库，并跳转 `/my-reviews`。
